内嵌表值函数

内嵌表值函数返回的是一个表,内嵌表值函数的功能相当于一个参数化的视图,无由BEGIN…END包括起来的函数体,
内嵌表值函数本质上是从数据库中筛选出的行组成的表,数据存在数据库中,可以通过内嵌表值函数修改数据库中的数据.

多语句表值函数

多语句表值函数的返回值是一个表,并且有一个由BEGIN…END语句括起来的函数体
区别于内嵌表值函数:多语句表值函数新建一个表用来存储查询到的结果集,查询的字段数量和类型要和新建表的字段相符.

SQL_92的游标语法规则

DECLARE cursor_name[INSENSITIVE][SCROLL]CURSOR
FOR select_statment
[FOR{READ ONLY | UPDATE [OF column_name[,..n]]}]
1.INSENSITIVE
游标选出的数据会存放在tempdb数据库下的一个临时表中,游标的读取操作都只对临时表操作,游标与基础表中对数据的操作彼此不影响.
2.SCROLL
表明所有读取操作(FIRST,LAST,PRIOR,NEXT,RELATIVE,ABSOLUTE)都可用.若不使用SCORLL关键字,则默认只能进行NEXT操作
3.READ ONLY
表明游标内的数据不能修改,只读,默认可修改
4.UPDATE
指明可修改的字段

SQL SERVER 提供的扩展游标声明语法

DECLARE cursor_name CURSOR
[LOCAL | GLOBAL]
[FORWARD_ONLY | SCROLL]
[STATIC | KEYSET | DYNAMIC | FAST_FORWARD]
[READ_ONLY | SCROLL_LOCKS | OPTIMISTIC]
[TYPE_WARNING]
FOR select_statment
[FOR{UPDATE [OF column_name[,..n]]}]
1.LOCAL 默认使用LOCAL
定义游标的作用域仅限在其所在的存储过程,触发器,或批处理中.
2.GLOBAL
游标的作用域是整个会话层,会话层指的是从用户连接到断开这段时间
3.FORWARD_ONLY
顺序操作,读取操作只能使用FETCH NEXT
若使用STATIC KEYSET DYNAMIC,则变成了SCORLL游标,否则默认使用FORWORD_ONLY
4.STATIC
与INSENSITIVE一样
5.KEYSET
6.DYNAMIC
指明游标中的数据会随着表中数据的变化而变化
7.FAST_FORWARD
顺序操作且只读

1.游标名称不区分大小写
2.游标语法基本分为 读/写操作区分 和 操作顺序 两类

触发器

函数和存储过程的区别

DML触发器

DML触发器是当数据库服务器中发生数据库操作语言(DML)事件时会自动执行的存储过程

AFTER触发器

在记录已经改变完之后才会被激活执行,主要用于记录变更后的处理或检查,一旦发现错误,也可以用POLLBACK TRANSACTION语句来回滚本次的操作

INSTEAD OF触发器

不同于AFTER触发器,一般用来取代原本的操作,在记录变更之前发生的,并不去执行原来的SQL语句里的操作(INSERT,UPDATE,DELETE),而是去执行触发器本身所定义的操作

DML触发器的作用

  1. 强制比CHECK约束更复杂的数据的完整性
  2. 使用自定义的错误提示信息
  3. 实现数据库中多张表的级联修改
  4. 比较数据库修改前后数据的状态
  5. 调用存储过程
  6. 维护非规范化数据

    DDL触发器

    DDL触发器是响应CREATE,ALTER,DROP,GRANT,DENY,REVOKE和UPDATE STATISTICS等语句而激活的.

创建DML触发器

DML触发器定义了两个特殊的表,一个是插入表,另一个是删除表,这两个表是建在数据库服务器的内存中的,是由系统管理的逻辑表,而不是真正存储在数据库中的物理表。

对于inserted和deleted表,用户只有读取数据的权限,没有修改数据的权限

inserted和deleted表只是存储着被操作数据操作前和操作后的状态,并非整个表的数据
